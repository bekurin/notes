## 1. connection에 대한 timeout을 설정하는 것은 클라이언트에서만 강제할 수 있다.

### 아무개의 이야기
spring boot를 사용하여 서버 개발을 하던 도중 너무 긴 컴퓨터 연산을 필요로 하는 http 요청에 대해서 connection을 끊어버렸으면 좋겠다는 지시를 받게 된다...

그렇게 http connection timeout에 대해 조사하게 되는데...

이유: client만이 server에 요청하는 시작 시간을 인지할 수 있다. 
Ex. if connection timeout is 30 sec
Connection start time: 00:00:00 -\> Request received time: 00:10:00
Client’s timeout time -\> 00:30:00
Server’s timeout time -\>00:40:00

Bard comment:
	연결 시간 초과는 클라이언트가 서버에 연결을 시작하기 때문에 클라이언트가 제어합니다. 클라이언트는 연결이 설정될 때까지 기다리려는 시간을 지정하는 시간 초과 값을 설정합니다. 시간 초과 기간 내에 연결이 설정되지 않으면 클라이언트는 연결 시도를 중단합니다.
	
	클라이언트가 연결 시간 초과를 제어하려는 몇 가지 이유가 있습니다. 첫째, 서버가 사용 불가능하거나 과부하된 경우 클라이언트가 무기한으로 차단되는 것을 방지할 수 있습니다. 둘째, 연결이 설정되지 않을 연결을 기다리는 것을 방지하여 클라이언트의 성능을 향상시킬 수 있습니다.
	
	클라이언트는 사용 중인 네트워크 프로토콜에 따라 다양한 방법으로 연결 시간 초과 값을 설정할 수 있습니다. 예를 들어 HTTP 프로토콜에서 연결 시간 초과 값은 Connection: timeout 헤더를 사용하여 설정할 수 있습니다.
	
	클라이언트가 연결 시간 초과를 제어하는 ​​이점 중 일부는 다음과 같습니다.
	
	서버가 사용 불가능하거나 과부하된 경우 클라이언트가 무기한으로 차단되는 것을 방지합니다.
	연결이 설정되지 않을 연결을 기다리는 것을 방지하여 클라이언트의 성능을 향상시킵니다.
	클라이언트가 할당된 대역폭을 초과하지 않도록 합니다.
	클라이언트가 연결 시간 초과를 제어하는 ​​단점 중 일부는 다음과 같습니다.
	
	연결 시간 초과 값이 너무 짧으면 클라이언트가 성공적으로 설정될 수 있는 연결을 중단할 수 있습니다.
	연결 시간 초과 값이 너무 길면 클라이언트가 연결이 설정되지 않을 연결을 기다리면서 리소스를 낭비할 수 있습니다.
	특정 응용 프로그램에 적합한 연결 시간 초과 값을 선택하는 것이 중요합니다. 최적의 값은 사용되는 네트워크 유형, 서버의 예상 부하 및 원하는 성능 수준과 같은 요인에 따라 다릅니다.
